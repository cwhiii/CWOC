What is CWOC?

C.W.'s Omni Chits is a personal task, note, and calendar management app. The core concept is a chit — a flexible record that can be a task, note, calendar event, alarm, checklist, or project, all in one data model.

Chits

A chit is the universal unit. What makes it appear in different views is which fields you fill in:

  • Calendar — has a start/end date or due date
  • Tasks — has a status (ToDo, In Progress, Blocked, Complete)
  • Notes — has note content
  • Checklists — has checklist items
  • Alerts — has alarms, timers, or notifications
  • Projects — marked as a "Project Master" with child chits

A single chit can appear in multiple views if it has the relevant fields.

Views & Tabs

The top tab bar switches between six views: Calendar, Checklists, Alerts, Projects, Tasks, Notes, plus a Global Search tab (magnifying glass icon). The sidebar (toggle by clicking the logo or the ☰ hamburger on mobile) provides date navigation, sorting, period selection, and filters.

Sidebar Layout

The sidebar is organized top-to-bottom:

  • Date Navigation — Year/month display, ◄/► arrows to move between periods, and a compact 📍 Today button to jump back to the current date
  • Create Chit — Directly below the date arrows. Click to create, middle-click to open in a new tab
  • Order — Sort by title, dates, status, manual drag, random, or upcoming
  • Period — Switch calendar period (Itinerary, Day, Week, Work Hours, X Days, Month, Year)
  • Filters — Collapsible filter groups for text, status, priority, tags, people, and show/hide options
  • People & Settings — Navigation to People and Settings pages
  • Clock, Weather, Reference, Help — Four compact buttons at the bottom. Clock and Weather open modals (Shift+click Weather for the full page). Reference and Help open their respective overlays/pages

The Alerts and Projects tabs have a View Mode toggle in the sidebar. Projects switches between List and Kanban; Alerts switches between Chits (showing chits that have alerts) and Independent (a 3-column board for quick alarms, timers, and stopwatches not tied to any chit).

Calendar View

Periods: Itinerary, Day, Week, Work Hours, X Days, Month, Year. Switch with the Period dropdown or . hotkey.

  • Start & end times → timed blocks
  • Due date only → ⌚ icon
  • All-day → all-day bar at top
  • Work Hours → only configured working hours and days
  • X Days → configurable count (Settings → Period Options)

Mobile Touch

On mobile devices, touch interactions use a unified tap-and-hold gesture system. All views share the same timing so behavior is consistent everywhere:

  • Tap — Normal tap/click behavior (open links, toggle checkboxes)
  • Hold ~0.4s then move — Drag to reorder. Hold your finger on a card or checklist item until you feel a short vibration, then drag. If you move before the hold completes, the page scrolls normally.
  • Hold ~1.2s without moving — Opens Quick Edit modal (or inline edit on Notes). A double vibration confirms the long-press activated. Once you start moving (dragging), long-press is cancelled.
  • Double-tap — Open chit in editor
  • Note previews — Tap "show more…" to expand truncated note text on cards
  • Kanban (Projects) — Hold a card then drag it to a different status column. Works in both the dashboard kanban view and the editor's project zone.

Hierarchical labels with colors. Nesting via / (e.g., Work/Projects/CWOC). Child tags inherit parent color. ★ Favorites appear at top.

Global Search

Press G or click the magnifying glass tab (last in the C CAPTN tab bar) to open the Global Search view. Unlike the sidebar filter, Global Search queries ALL fields across ALL chits regardless of which tab they belong to.

  • Search Input + Go — Type a query in the search input and press Enter or click Go. The search performs case-insensitive substring matching across every chit field (title, note, tags, status, priority, location, people, checklist items, dates, color, alerts, and more).
  • Result Cards — Each matching chit is displayed as a card showing the chit title and every matched field with the matching text highlighted. Click a result card to open that chit in the editor.
  • Sidebar Filters — Sidebar filters (Status, Priority, Tags, etc.) remain active during Global Search, narrowing results further.

Filtering & Sorting

The sidebar provides multiple filter groups, each collapsible by clicking its label:

  • Filter Text — Free-text search across title, notes, tags, status, people, location, and priority
  • Status — Multi-select: Any, ToDo, In Progress, Blocked, Complete
  • Priority — Multi-select: Any, Low, Medium, High
  • Tags — Multi-select with search. Favorites appear first
  • People — Multi-select from contacts assigned to chits
  • Show — Toggle visibility of Pinned, Archived, Unmarked chits, and Hide Past-Due

When any filter is active, a "✕ Clear All" button appears. If the current tab has a default filter (configured in Settings → Default Filters), a "🔄 Defaults" button resets to that filter.

Sorting

The Order section in the sidebar provides sort options: Title, Start Date, Due Date, Updated, Created, Status, Manual (drag-to-reorder), Random/Shuffle, and Upcoming (due soon first, completed at bottom). Click the ▲/▼ button to toggle ascending/descending.

Notes View

Multi-column masonry layout with full rendered markdown. [[chit title]] auto-links to other chits.

Shift+click (or very long-press ~1.2s on mobile) a note card to edit it in place. A short hold (~0.4s) then drag lets you reorder notes between columns. While editing, drag-and-drop reordering is disabled so you can click to position the cursor. Press ESC or click outside to save and exit edit mode.

Clocks

Press L on the dashboard or click the 🕐 Clock button in the sidebar to open the clocks modal. Configure which clocks are displayed and their layout in Settings → Clocks.

  • 24 Hour — Standard 24-hour digital display (HH:MM:SS)
  • 12 Hour — 12-hour digital display with AM/PM
  • 12 Hour Analog — Traditional analog clock face with hour, minute, and second hands
  • HST — Holeman Simplified Time. Represents the day as 0–100 sentidays (sd) shown on a progress bar. Midnight = 0.000 sd, noon = 50.000 sd, end of day = 100.000 sd. "Sentiday" is the Holeman Simplified English spelling of centiday.

Drag clocks between Active and Inactive zones in Settings to choose which appear. Use "Change Orientation" to switch between horizontal and vertical layout.

Weather

Press W on the dashboard to open the weather modal. It shows the current forecast for your default saved location, including weather icon, high/low temperatures (°F), and precipitation. If no default location is configured, the modal will prompt you to set one in Settings. The modal includes a 📊 Full Forecast button that opens the dedicated Weather page.

In the chit editor, weather loads automatically when a chit has both a location and a date. Stored forecast data displays immediately on load, then a live fetch updates it in the background. Placeholder messages appear when either location or date is missing.

Weather Data on Chits

Each chit with a location and date stores its own weather forecast. The stored data includes:

  • High / Low — Forecast high and low temperatures (stored in Celsius, displayed in °F)
  • Precipitation — Expected precipitation sum in mm
  • Weather Code — Determines the weather icon (☀️ sunny, 🌧️ rainy, 🌨️ snowy, etc.)
  • Focus Date — The date the forecast applies to (earliest of start or due date)
  • Updated Time — When the forecast was last refreshed

Weather data is updated automatically by the server — no browser needs to be open. When you save a chit in the editor, any fetched weather data is saved with it.

Calendar View Weather Display

On the calendar, chits with stored weather data show a compact weather summary on their event card:

  • Weather icon — Derived from the weather code (e.g., ☀️, 🌤️, 🌧️, 🌨️)
  • High / Low temps — Displayed in °F
  • Precipitation — Shown on Day, Week, Work Hours, and X Days views
  • ⏳ Stale indicator — Appears when the forecast data is more than 24 hours old

If a chit has a location but no stored weather data yet, it falls back to fetching weather in the background. The 🌤️ Weather button in the sidebar opens the weather modal (click) or navigates to the full Weather page (Shift+click).

Weather Page

The Weather page shows a comprehensive forecast table for all your saved locations. Access it via:

  • Shift+W from the dashboard
  • The 📊 Full Forecast button in the weather modal
  • Shift+click the 🌤️ Weather button in the sidebar

The page displays one row per saved location, with up to 16 days of forecast data. Each day block shows the weather icon, high/low temperatures in °F, and precipitation. Drag rows by the ☰ handle to reorder locations — your arrangement is saved automatically. Days where you have a chit at that location are highlighted with a darker background. Click any day block to jump to that day in the Calendar Day view — any chits at that location will flash briefly to draw your eye. If no saved locations are configured, a message directs you to add them in Settings. Press ESC to navigate back.

City Rows from Chits

Below the saved-location rows, the Weather page automatically adds rows for cities where you have upcoming chits but no saved location. These rows are labeled with the city name (extracted from the chit's address) and only show weather blocks on days that have chits in that city — other days are left empty. Click a day block to jump to that day in the Calendar, same as saved-location rows.

Weather Update Schedule

The server automatically refreshes weather forecasts on two schedules:

  • Hourly — Chits with dates in the next 7 days are updated every 60 minutes for higher accuracy
  • Daily — Chits with dates 8–16 days out are updated once every 24 hours for a general outlook

Updates run in the background on the server. Each unique location is geocoded once and reused for all chits at that address. You can also trigger a manual update via the API.

Saved Locations

Manage saved locations in Settings → Saved Locations. Each location has a label and an address. Use the radio button to mark one as the default.

  • Adding — Click ➕ Add Location to add a new row. Fill in a label and address.
  • Default — Select the radio button next to a location to make it the default. If only one location has an address, it is auto-selected.
  • Removing — Click ✕ to remove a row. At least one row always remains.
  • Saving — Empty-address rows are cleaned up on save.

Saved locations are used by the editor's Location zone dropdown, the compact 📍 dropdown in the title area, the +Location button, and the dashboard weather modal (W).

Chit Editor

Collapsible zones: Title, Dates & Times, Task, Location, Tags, People, Notes, Checklist, Alerts, Health Indicators, Color, Projects, Habit Log. An "📜 Audit Log" button at the bottom links to the filtered audit log for the current chit. 📱 QR button generates data or link QR codes.

When creating a new chit, only the zone relevant to the current view is expanded (e.g., Notes zone when coming from the Notes tab). All other zones start collapsed. The title field is auto-focused. If you have a default saved location, it's auto-applied to new chits.

Alerts Zone

Add notifications, alarms, timers, and stopwatches. Each alarm has a time, days, and optional "Delete chit after dismissal" checkbox — when checked, dismissing the alarm soft-deletes the chit.

Notifications fire relative to a chit's start or due time. Each notification has a combined timing dropdown (before start, after start, before due, after due) to precisely control when the notification fires. Enable the 🔁 loop toggle to repeat the notification every 60 seconds until acknowledged.

Alarms trigger system notifications with sound on all platforms (macOS, Linux, Android) via the browser Notification API. On Android and Linux Firefox, the device will also vibrate. If sound cannot autoplay due to browser restrictions, a tap or keypress will unlock audio playback. Grant notification permission when prompted for full functionality.

Independent Alerts Board

Switch to the Independent view mode in the Alarms tab sidebar to access a 3-column board for quick alarms, timers, and stopwatches that aren't connected to any chit. Click the + button in each column header to add a new item. Edit names, times, and durations inline — changes save automatically on blur. Delete items with the ❌ button. Timers and stopwatches have Start/Pause, Reset, and Lap controls just like in the chit editor. Timers and stopwatches start immediately when created — no save step needed.

Task Zone

Status (ToDo, In Progress, Blocked, Complete), Priority (High, Medium, Low), and Severity (Critical, Major, Normal, Minor). When a chit is shared with other users, an Assignee dropdown appears after Severity, listing only the users the chit is currently shared with. Removing a shared user from the People zone automatically updates the Assignee options.

In the Tasks view, status is shown with matching icons: ● ToDo, ⟳ In Progress, ⊘ Blocked, ✓ Complete.

Location Zone

  • +Location — Populates the address from your default saved location
  • Saved Locations dropdown — Select any saved location by label to populate the address
  • ✕ Clear — Clears the address, map, and weather display
  • 📍 compact dropdown — Quick-access dropdown in the title/weather area for selecting a saved location without scrolling to the Location zone
  • Google Maps button — When "Prefer Google for Maps" is enabled in Settings → Chit Options, the Map and Directions buttons use Google Maps instead of OpenStreetMap. No API key or account needed — these are plain web links.

A chit can have at most one location. Weather and map load automatically when a location and date are both present.

People Zone

The People zone displays both contacts and system users in a grouped alphabetical tree. Contacts appear as plain chips with favorite stars (★). System users appear with a icon and an inline pill toggle offering two roles: 👁️ Viewer and ✏️ Manager.

  • Search — The search bar searches across all contact fields: name, email, phone, address, organization, social context, call signs, X handles, websites, notes, and tags
  • Adding a contact — Click an unlinked contact chip to add them to the chit's people list
  • Adding a share — Click an unshared system user chip to add them to the chit's shares list as a Viewer
  • Changing role — Click the pill toggle on a shared user to switch between Viewer and Manager
  • Removing a share — Click the ✕ button on a shared user chip to remove them from the shares list
  • Owner excluded — The currently authenticated user (the chit owner) does not appear in the system user list
  • 🥷 Stealth toggle — At the bottom of the People zone, a stealth checkbox hides the chit from all other users. Visible to owners and managers only; hidden for viewers

People Expand View

Click the "Expand" button in the People zone header to open a full-screen table view. All controls are on the left side of each row; an edit/view button is on the right.

  • Table columns — Controls, Icon, Thumbnail, Name, Email, Org, Notes, Status, Edit — with a sticky header row
  • Full-field search — Search across all contact and user fields to find people to add
  • Favorites section — Favorite contacts appear in a highlighted "★ Favorites" section at the top
  • Controls (left) — +/✕ add/remove, Viewer/Manager pill toggle, 📌 assign, RSVP badges — all grouped on the left
  • Edit (right) — Opens the contact editor or user profile in a new tab. User profiles open in read-only mode unless it's your own
  • User icons — System users display a icon to distinguish them from contacts

User Profiles

User profiles use the same editor page as contacts. Clicking a system user in the People expand view or the Rolodex opens the contact editor in profile mode (read-only for other users, fully editable for your own). Profile mode adds Account and Change Password zones, and hides contact-only features (favorite, tags, delete, QR).

Users can pick a profile color from the standard palette or any custom colors defined in Settings. This color appears on their chips in the editor People zone, the sidebar People filter, and tints the editor background.

User chips are visually distinguished from contact chips by a thicker, very dark brown border and a icon placeholder (when no profile image is set). The sidebar People filter includes both contacts and system users.

Managers have the same sharing-management permissions as owners — they can add and remove shared users, change roles, and toggle stealth. Viewers see the People zone in read-only mode with no toggles or stealth control.

Owner Chip

The chit owner's display name is shown as a chip in the title zone header, inline with the "Title" label and the pinned icon. The chip uses the same visual style as people chips (background color, thumbnail image or placeholder). The owner chip is always displayed, even when the current user is the owner.

Quick Edit Modal

Shift+click any chit. Edit title (double-click), change Priority/Severity/Status, Pin/Archive/Delete/QR. Recurring chits get per-field "All" checkboxes and series edit options.

Settings

  • General — Username, time format (24h, 12h, HST), sex toggle, snooze length, calendar snapping
  • Period Options — Week start day, view hours range, scroll-to hour, then enabled periods with per-period options (X Days count, Work Days and Work Hours) nested directly below their checkboxes
  • Tag Editor — Tree view, create/edit/delete/favorite, "+" for child tags
  • Saved Locations — Add, label, and set a default location for use across the editor and dashboard weather
  • Chit Options — Fade past, highlight overdue, delete past alarms, show tab counts, prefer Google for Maps
  • Custom Colors — Two sections: Default Colors (7 palette colors plus Overdue Border and Blocked Border swatches) and Custom Colors (user-added with delete buttons). Click any swatch to assign it as the overdue or blocked border color. Ring indicators show which color is assigned to each role. The overdue border highlights chits past their due date; the blocked border highlights chits with Blocked status.
  • Clocks — Choose active clocks, drag to reorder, set orientation (horizontal/vertical)
  • Default Filters — Per-tab default search text

  • Version & Updates — Current version, upgrade button, update log

Data Management

The Settings page includes a 📦 Data Management box for exporting and importing your data as JSON files. There are two data categories, each with its own Export and Import buttons:

Chit Data

Exports all chits (including deleted chits) as a JSON file. This captures every field on every chit — titles, notes, dates, tags, checklists, alerts, recurrence rules, and more.

User Data

Exports your settings, tags, custom colors, saved locations, and contacts as a JSON file. Use this to back up your configuration or replicate it on another CWOC instance.

Import Modes

When importing a file, you choose one of two modes:

  • ➕ Add — Merges the imported data with your existing data. Nothing is removed. For chits, each imported chit is added as a new record. For user data, array fields (tags, colors, locations) are merged and deduplicated, and contacts are added unless an exact name match already exists.
  • 🔄 Replace — Deletes all existing data of that type and replaces it with the imported data. A confirmation dialog appears before any data is removed.

File Format

Exported files are self-contained JSON with metadata including the CWOC version, export timestamp, and instance ID. Files can be imported into any CWOC instance.

Visual Indicators

Visual indicators are small icons displayed alongside chit titles in all views to signal key properties. Configure their visibility in Settings → Visual Indicators. Each indicator has three visibility modes:

  • Always — The indicator icon is always shown
  • Never — The indicator icon is never shown
  • If Space — Show the icon on card views and calendar day/week slots, but hide on month cells where space is tight

Indicator Types

  • 🔔 Alarm — Chit has alarm alerts
  • 📢 Notification — Chit has notification alerts
  • ⏱️ Timer — Chit has timer alerts
  • ⏲️ Stopwatch — Chit has stopwatch alerts
  • 🌤️ Weather — Chit has a location with weather data
  • 👥 People — Chit has people assigned
  • ❤️ Health — Chit has health indicator data

Combine Alerts Toggle

The Combine Alerts checkbox in Visual Indicators switches between two display modes:

  • Combined mode (checked) — A single 🛎️ icon represents all alert types. A "Combined Alerts" dropdown controls visibility (Always / Never / If Space).
  • Individual mode (unchecked) — Separate icons appear per alert type, each with its own visibility dropdown: 🔔 Alarm, 📢 Notification, ⏱️ Timer, ⏲️ Stopwatch.

Calendar Views

Calendar views (Week, Day, Month, Itinerary, Seven-Day, X-Day) always use combined mode for alerts since calendar cells are space-constrained. Weather, People, and Health indicators still respect their individual settings.

Card Views

Checklists, Tasks, Notes, Alarms, and Projects views respect the Combine Alerts toggle — showing either one combined 🛎️ icon or individual per-type icons depending on the setting.

"If Space" Behavior

When a visibility dropdown is set to "If Space," indicators always appear on card views and calendar day/week slots (sufficient room). On month cells, indicators may be hidden when the title would be truncated.

Version Management

The Settings page includes a Version & Updates section that shows the currently installed version number and the date/time it was last updated.

  • Version Info — Displays the current version and "Last Updated" timestamp. If no version info is available, a fallback message is shown.
  • Upgrade Omni Chits — Click this button to trigger an update. A modal opens showing real-time output from the update script.
  • Update Modal — Streams the update log in a terminal-style display. Log lines are color-coded by prefix: [OK] green, [WARN] yellow, [ERROR] red, [STEP] blue. The Close button is enabled once the update finishes.

Trash

When you delete a chit, it is soft-deleted and moved to the Trash. You can access the Trash page from the navigation header or from Settings → Data.

  • User scoping — Regular users see only their own deleted chits. Admins see all deleted chits across all users, with an extra "Owner" column showing who each chit belongs to.
  • Restore — Click the Restore button on any row to un-delete the chit and return it to your active views.
  • Permanent delete — Click the Delete button to permanently remove a chit from the database. This cannot be undone.
  • Bulk actions — Use the checkboxes to select multiple chits, then use the bulk Restore or Delete buttons in the toolbar.

Audit Log

CWOC tracks all changes to your data with a built-in audit log. Every create, update, and delete operation on chits, contacts, settings, and system upgrades is recorded automatically.

What Is Tracked

  • Chits — Created, updated, and deleted chits
  • Contacts — Created, updated, and deleted contacts
  • Independent Alerts — Created, updated, and deleted independent alarms, timers, and stopwatches
  • Settings — Any changes to your configuration
  • System — Application upgrades (old and new version recorded)

Each audit entry records the actor (who made the change), a timestamp, the action performed, and for updates, a field-level diff showing old and new values for each changed field.

Actor

The actor field reflects the username configured in Settings → General. If no username is set, the actor defaults to "Unknown Gremlin". The username is stored as a snapshot at the time of the change — if you later change your username, older entries still show the name you had when the change was made.

Accessing the Audit Log

  • Global Audit Log page — Access via Settings → Data → Audit Log, or the Audit Log link in the navigation header. Shows all audit entries across all entities.
  • Per-chit link — The chit editor has an "📜 Audit Log" button at the bottom that deep-links to the audit log filtered for that chit.
  • Per-contact link — The contact editor has an "📜 Audit Log" button that deep-links to the audit log filtered for that contact.

Global Audit Log Page

The dedicated audit log page provides a full-featured interface for browsing change history:

  • Real-time filtering — Filters apply immediately as you change entity type, actor, date range (Start/End), or page size. No "Apply" button needed.
  • Sorting — Click column headers (Time, Entity, Actor, Action, Entity Type) to toggle ascending/descending sort
  • Entity links — The Entity column links directly to the chit editor, contact editor, settings page, or trash (for deleted chits)
  • Pagination — Configurable page size (25 / 50 / 100 / 500) with a "Load More" button
  • Change details — Expand any entry to see which fields changed, with old → new values

Log Management

  • Delete Audit Logs — Opens a modal to prune entries older than a selected timeframe (Past Hour / Day / Week / Month / Year / All Time). Shows the count of entries to be deleted. Option to download CSV before deleting.
  • CSV Export — Downloads the currently filtered entries as a CSV file

Audit Log Limits

In Settings → Data → Audit Log Limits, you can configure automatic pruning:

  • Enable Pruning — Checkbox to enable or disable automatic pruning. When unchecked, both limit inputs are disabled and pruning is skipped entirely.
  • Max Age (days) — Entries older than this are pruned automatically. Default: 1096 (3 years). Minimum: 1.
  • Max Size (MB) — When the audit log exceeds this size (whole integers only), oldest entries are pruned. Default: 1 MB. Minimum: 1.

Auto-pruning runs at startup and after saving settings. Uncheck "Enable Pruning" to disable all automatic pruning.

Column Reordering

On the audit log page, you can drag and drop column headers to reorder them. The detail toggle column (first column) is fixed. Column order is not persisted — it resets on page reload.

Habits View

CWOC supports explicit habit tracking via an opt-in model. Any chit can become a habit by checking the "Track as habit" checkbox in the editor's Dates zone. Habits have their own goal/progress system, charts, and a dedicated Habits view in the Tasks tab.

Marking a Chit as a Habit

The "🎯 Habit" button is always visible in the editor's Dates zone header. When you click it:

  • Auto-enables Repeat — If the chit doesn't already have a recurrence rule, Repeat is automatically turned on with a default of Daily
  • Locks Repeat — While habit is active, the Repeat checkbox is locked on (you must turn off the habit first to disable Repeat)
  • Reveals habit controls — A Goal input and progress display (X / Y) appear on a dedicated row. A separate "📅 Calendar" row shows the "Show on calendar" toggle
  • Forces All Day — Habits are always all-day events. The "📅 All Day" button shows active and is disabled while habit is on
  • Shows 🎯 icon — Habit chits display the 🎯 target icon instead of the 🔁 repeat icon everywhere: editor title bar, calendar events, quick-edit modal, and tooltips
  • Auto-tags — The system automatically adds Habits and Habits/[title] tags to the chit

Goal and Progress

Each habit has a Goal (how many completions per period you're aiming for, minimum 1) and a Progress count (how many you've done this period). When progress reaches the goal, the chit is automatically marked Complete. Progress is capped at the goal value and can never exceed it.

Switching to Habits Mode

When the Tasks tab is active, the sidebar shows a View Mode section with three buttons: 📋 Tasks, 🎯 Habits, and 📌 Assigned. Click "Habits" to enter Habits mode. Your selection is remembered across sessions. Only chits explicitly marked with "Track as habit" appear here — not all recurring chits.

Habit Cards

Each habit appears as a card showing:

  • Progress — Displayed as "X / Y" where X is the current period's completion count and Y is the goal
  • Interaction — For goal=1 habits, a checkbox toggles complete/incomplete. For goal>1 habits, +/− buttons increment or decrement the count
  • Frequency label — The recurrence pattern (e.g., "Every day", "Every week on Mon, Wed, Fri")
  • Streak — A 🔥 flame icon followed by the count of consecutive periods where the goal was met, walking backward from the most recent past period. Broken-off (skipped) periods are neutral and don't break the streak
  • Cycle progress — A teal badge showing the percentage of the way to the goal this period (e.g., 4/5 = 80%)
  • Overall success rate — A brown badge showing the percentage of past cycles where the goal was fully met, within the configured success window
  • Status badge — Shows the current status when set
  • Title — Double-click or long-press to open in the editor, matching standard interaction patterns

Period Rollover

Habits reset automatically at period boundaries. When a new period starts (e.g., a new day for daily habits, a new week for weekly), the system snapshots the previous period's progress into the recurrence exceptions history, resets progress to 0, and clears the Complete status. Rollover is evaluated lazily — when you open the Habits view or the editor, not via a background process.

Habit Log (Editor)

The chit editor includes a Habit Log collapsible zone (visible only when "Track as habit" is checked). It shows:

  • Period history — Past periods listed in reverse chronological order with their completion counts (X / Y). Click any count to edit it inline — useful for correcting mistakes or filling in data you forgot to record
  • Charts — Three canvas-based charts in a 2-column grid: completion over time (bar chart), success rate trend (line chart), and streak history (timeline). Charts update when you edit past period counts. The time range matches your Success Rate Window setting

Calendar Visibility

Each habit has a "Show on calendar" toggle. When off, the habit is hidden from all calendar views but still appears in the Habits view. The default value for new habits is controlled by the "Default: show habits on calendar" setting in Settings → Habits.

Sorting

Habit cards are sorted with incomplete habits first and completed habits pushed to the bottom. Within each group, the existing sort and filter settings are preserved.

Settings

The Settings page includes a Habits section with:

  • Success rate window — Controls the rolling time window for success rate and chart calculations:
    • Last 7 days — Only periods in the past week
    • Last 30 days — Past month (default)
    • Last 90 days — Past quarter
    • All time — Every period since habit tracking started
  • Default: show habits on calendar — Toggle controlling the default value of "Show on calendar" for newly created habits

Contact Editor

Contacts have name fields (prefix, given, middle, surname, suffix), phone, email, address, social/web, security (Signal, PGP), and context (organization, social context).

  • Notes — Markdown notes field for free-form text about the contact
  • Tags — Tags for contacts, auto-prefixed with "Contact/" (shared with the general tag system)
  • Suffixes — Text suffixes (Jr., Sr., Esq., Ph.D., M.D.) listed before numeric suffixes (I–X)

Sharing

Chits can be shared with other system users, giving them Viewer or Manager access. All sharing controls live in the People zone of the chit editor.

Roles

  • Owner — The user who created the chit. Full control over all fields and sharing
  • Manager — Can edit the chit and manage sharing (add/remove users, change roles, toggle stealth)
  • Viewer — Can view the chit but cannot change sharing settings. The People zone appears in read-only mode

Dashboard

Shared chits display a 🔗 icon on their dashboard card. Hover over the icon to see a tooltip with the owner's name, the list of shared users and their roles, and your effective role. Non-shared chits show no icon.

Kiosk

The Kiosk is a read-only, unauthenticated display designed for wall-mounted screens or shared monitors. It shows calendar events and active tasks filtered by the tags you choose — no login required.

Setting Up the Kiosk

Configure which tags appear on the kiosk in Settings → Tools → Kiosk:

  • Tag picker — A tree of your user-created tags (system tags are excluded). Check the tags you want displayed on the kiosk. Selecting a parent tag automatically includes all child tags.
  • Save — Click Save to persist your kiosk tag selection. This is required for direct-URL access to work.
  • 📺 Open Kiosk → — Opens the kiosk page with the currently selected tags.

Accessing the Kiosk

  • Direct URL — Navigate to /kiosk on your server. The page loads the saved tag configuration automatically — no login needed. Ideal for bookmarking on a wall display.
  • With specific tags — Append tags to the URL: /kiosk?tags=Work,Personal. URL tags override the saved configuration.
  • Launch button — Use the "📺 Open Kiosk →" button in Settings to open with the currently selected tags.

Kiosk Display

The kiosk shows a two-column layout:

  • Calendar (left) — Upcoming events grouped by day, with time, title, color, and owner name. Today is highlighted with a gold accent.
  • Tasks (right) — Active tasks (not complete) sorted by status: In Progress first, then Blocked, then ToDo. Each shows status icon, title, due date, and owner.

Period Navigation

A toolbar at the top lets you switch between Day, Week, and Month views, navigate forward/backward, and jump to Today. The week start day respects your Settings configuration.

Auto-Refresh & Interaction

  • Auto-refresh — The kiosk refreshes data every 60 seconds automatically.
  • Click to edit — Clicking any event or task opens it in the chit editor. After saving, you're returned to the kiosk.

A tag legend at the bottom shows which tags are active, and a timestamp shows when data was last refreshed.

Network Access

The Settings page includes a 🌐 Network Access block (admin only) for configuring external network access providers. Currently supports Tailscale, a WireGuard-based mesh VPN for secure remote access without port forwarding.

Tailscale

  • Status — Shows the current Tailscale state: ⚪ Not Installed, 🟡 Inactive, 🟢 Connected (with IP and hostname), or 🔴 Error. Click 🔄 Check Status to re-check (read-only — does not attempt to reconnect).
  • Auth Key — Enter your Tailscale pre-authentication key (from the Tailscale admin console). The key is masked by default; click 👁️ to reveal it. Click 🔑 Get Key to open the Tailscale admin console.
  • Enabled — Toggle whether Tailscale should be active.
  • 💾 Save Config — Saves the auth key and enabled state immediately (independent of the main settings Save button). If the auth key changes, Tailscale is automatically disconnected and logged out so the old credentials are purged.
  • ▶️ Connect — Authenticates and connects Tailscale using the saved auth key. Forces re-authentication each time.
  • ⏹️ Disconnect — Stops the Tailscale connection.

Troubleshooting

  • TUN device missing — If you see "TUN device not available (/dev/net/tun missing)", your server is likely running in a Proxmox LXC container that doesn't have TUN access enabled. To fix, add these two lines to the container config on the Proxmox host at /etc/pve/lxc/<CTID>.conf (where CTID is the numeric container ID, e.g. 111): 
    lxc.cgroup2.devices.allow: c 10:200 rwm
lxc.mount.entry: /dev/net/tun dev/net/tun none bind,create=file
    Then restart the container from Proxmox (pct stop <CTID> && pct start <CTID>).
  • Daemon won't start — Check the server logs with journalctl -xeu tailscaled.service for details.
  • Auth key expired — One-time keys can only be used once. Generate a new key from the Tailscale admin console and save it.
