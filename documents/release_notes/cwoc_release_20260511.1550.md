Point in Time chits now show on the calendar (all views: week, day, 7-day, month, itinerary) with a distinctive diamond/arrow shape — pointy angled ends on both left and right sides via clip-path. Emails now populate point_in_time instead of start_datetime. System tag computation updated so any chit with a point_in_time gets the Calendar tag. Point-in-time events cannot be resized (no resize handle) and participate in normal overlap layout. After dragging any event, past-event fading is re-applied so items moved behind the current time correctly go semitransparent.
