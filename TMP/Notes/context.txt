Functionality Overview: C.W.'s Omni Chits (CWOC) is a task and note management web application hosted on a Proxmox LXC container at http://192.168.1.111:3333. It allows users to create, edit, and delete "chits" (tasks or notes) with attributes like title, status, labels, due dates, checklists, and more. The app organizes chits into tabs such as Calendar, Checklists, Alarms, Projects, Tasks, and Notes, enabling users to filter and search through them, with a calendar view for time-based chits and visual elements like labels and pastel-colored events.

Code Flow - Backend: The backend, powered by FastAPI in /app/backend/main.py, initializes a SQLite database at /app/data/app.db using init_db() to create the chits table. FastAPI mounts /app/frontend at / for the main UI and /app/static at /static/ for assets like images. API endpoints handle chit operations: GET /api/chits fetches all non-deleted chits, POST /api/chits creates new chits with auto-generated system labels (e.g., "Calendar" for time-based chits), PUT /api/chits/{chit_id} updates chits, and DELETE /api/chits/{chit_id} soft-deletes them. The backend uses Pydantic models for validation and JSON serialization for fields like labels and checklists, but currently fails on /api/chits due to a runtime error.

Code Flow - Frontend: The frontend, served from /app/frontend/index.html, uses vanilla JavaScript to fetch chits via fetchChits() calling /api/chits, then renders them in the chit-list div using displayChits(). The Calendar tab displays a weekly view with all-day and timed events, calculating positions and overlaps for timed events using pixel-based styling (e.g., top and height for event duration). Other tabs (e.g., Checklists, Tasks) filter chits by labels, with search and status filters applied dynamically. Static assets like cwod_logo.png are referenced but fail to load due to a FastAPI static file mounting issue, breaking visual elements.
